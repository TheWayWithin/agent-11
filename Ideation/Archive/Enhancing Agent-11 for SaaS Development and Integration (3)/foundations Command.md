# /foundations Command

Manage foundation documents from BOS-AI and their integration with Agent-11.

## Usage

```
/foundations <subcommand>
```

## Subcommands

### `init`

Initialize and process foundation documents that have been copied to `documents/foundations/`.

**What it does:**
1. Scans `documents/foundations/` for all markdown files
2. Generates `documents/foundations/handoff-manifest.json` with file inventory and checksums
3. Creates context summaries in `.context/summaries/` for Rolling Wave Planning
4. Validates that required documents are present
5. Reports any missing or incomplete documents

**Example:**
```
/foundations init
```

### `refresh`

Regenerate context summaries from foundation documents (useful after manual edits).

**Example:**
```
/foundations refresh
```

### `validate`

Check that all foundation documents required by the current project plan are present and valid.

**Example:**
```
/foundations validate
```

## Integration with Rolling Wave Planning

The context summaries generated by `/foundations init` are designed to fit within the Rolling Wave Planning context budget:

| Summary | Source Document | Token Budget | Provides |
|---------|-----------------|--------------|----------|
| `vision-summary.md` | strategic-plan.md | ~100 tokens | Vision, mission, goals |
| `icp-summary.md` | client-success-blueprint.md | ~100 tokens | ICP, personas, pain points |
| `brand-summary.md` | brand-style-guidelines.md | ~100 tokens | Colors, fonts, tone |
| `prd-summary.md` | prd.md | ~200 tokens | Feature list, priorities |

**Total baseline context: ~500 tokens** (always loaded)

This leaves ample room for phase-specific context and conversation.
