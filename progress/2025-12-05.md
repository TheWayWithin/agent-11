## Root Cause Analysis

**Problem**: Users experiencing Claude repeating already-completed work when resuming sessions.

**Root Causes Identified**:
1. Phase transitions were advisory, not blocking
2. No session resumption staleness check
3. Pre-clear updates often skipped
4. No timestamp discipline for staleness detection
5. Handoff updates not mandatory

**Solution**: Move from advisory protocols to structural enforcement that makes missed updates nearly impossible.

## Lessons Learned

1. **Advisory protocols fail under cognitive load** - When coordinators are busy orchestrating complex missions, they naturally skip optional verification steps. Making them mandatory and blocking prevents this.

2. **Timestamps enable staleness detection** - Without timestamps, there's no way to programmatically detect if files are out of sync. ISO timestamps on all entries makes this possible.

3. **Session resumption is a critical moment** - The start of a new session is when stale file issues manifest. A mandatory staleness check at this point catches problems before work is repeated.

4. **Structural enforcement beats behavioral compliance** - Rather than asking Claude to remember to update files, making it impossible to proceed without updates ensures compliance.
