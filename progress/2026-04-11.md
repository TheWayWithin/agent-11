## Issues

### Voice guide kept getting auto-indexed as a skill
**Symptom**: the default voice guide file was showing up in Claude Code's command palette as `scripts:voice-guide-default` (first attempt) and later `data:voice-guide-default` (second attempt). Cosmetic noise on every repo it deployed to.
**Attempts**:
1. First home was `project/commands/scripts/voice-guide-default.md`, next to the old Python script. Palette showed `scripts:voice-guide-default`. Wrong.
2. Moved to `project/commands/data/voice-guide-default.md`, reasoning the `data/` prefix would exclude it. Palette showed `data:voice-guide-default`. Still wrong.
3. Moved to `project/data/voice-guide-default.md` (deploys to `.claude/data/`), outside `.claude/commands/` entirely. Palette stayed clean.
**Root cause**: Claude Code's harness walks the full tree under `.claude/commands/` and indexes every `.md` file it finds, regardless of subdirectory name. The file had to live outside the commands tree entirely.
**Learning**: subdirectory names are not a filter for harness indexing. Only the parent path counts.

### BOS-AI local install failed on first attempt
**Symptom**: ran agent-11's install.sh from the local working copy into BOS-AI. Script announced "Execution mode: remote", tried to download `blog.md` from GitHub, failed because nothing was pushed yet, rolled back cleanly.
**Root cause**: `PROJECT_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)"` on line 11 of install.sh. The script is at `project/deployment/scripts/install.sh` — three levels deep — so `../..` resolved to `agent-11/project/` and the `.git` check failed. Detection fell through to remote mode.
**Resolution**: changed `../..` to `../../..`. Re-ran the install. Worked end-to-end.
**Learning**: pre-existing bugs hide in codepaths nobody exercises. This one had been wrong since whenever install.sh was first written, but curl-piped remote installs (the common path) never go through the local-mode branch, so it never surfaced. First time anyone tried a local install with content that didn't exist on GitHub yet, it broke.

## Lessons Learned

- Opus / Sonnet 4.6 follows a 1500-word voice guide materially better than gpt-4o-mini. The smaller model would catch half the banned vocabulary and miss the other half. The larger model holds the rules cold.
- When a command has full edit control, a voice scrub should be a rewrite pass, not a warning. The old Python script could only flag AI-tell words after generation because it had no way to call back to gpt-4o-mini for a fix. Claude-native just rewrites the sentence on the spot.
- Physical file location matters more than directory naming conventions when the harness is auto-discovering things. Trust the path, not the subdirectory label.
